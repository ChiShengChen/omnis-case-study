## 五、核心問題：為什麼 IL 不是教科書上的 1.3%？

### 5.1 教科書公式 vs 現實

Uniswap V2 的 IL 公式（全範圍、靜態、不 rebalance）：

```
IL = 2√r / (1+r) - 1    其中 r = 新價格 / 舊價格
```

BTC 跌 26.7%，r = 0.733：

```
IL_v2 = 2√0.733 / (1+0.733) - 1 = -1.3%
```

**如果是 V2 full-range 靜態倉位，IL 只有 1.3%，大約 $10。** 這就是你說的「變動一倍才 5.7%」那個公式。

但 Jeff 實際的 IL 是 **12.7%（$104）**，是理論值的 **10 倍**。原因是兩層放大效應：

### 5.2 第一層放大：集中流動性 ≈ 16 倍槓桿

Steer vault 不是 V2 的「全範圍」部署。它把資金集中在當前價格附近大約 **±6% 的窄區間**（典型 1,200 tick 寬度）。

這等同於大約 **16 倍的資本效率**——同樣的資金提供了 16 倍的流動性深度。但 IL 也同步放大：

```
IL_concentrated ≈ IL_v2 × 集中倍數
               ≈ 1.3% × 16.7
               ≈ 21.7%（理論單次靜態集中倉位）
```

如果 vault 建好一個集中倉位然後放著不動，價格跌出區間後，倉位會變成 100% WBTC 停止運作，IL 就固定在那個數字。但 Steer vault 不是這樣做的。

### 5.3 第二層放大：Rebalance 造成「鎖虧→重新曝險」的循環

Steer vault 的核心邏輯是：**當價格偏離區間時，自動 rebalance 到新的集中區間**。

在單邊下跌中，這個機制變成一個不斷累積虧損的循環：

```
第 1 輪：
  BTC $93K → Vault 部署在 [$87K, $99K] 區間
  BTC 跌到 $87K → 倉位變成幾乎全是 WBTC（IL 已累積）
  → Vault 執行 rebalance：burn 舊倉位（鎖定虧損）

第 2 輪：
  → Vault 重新 mint 在 [$82K, $92K] 區間
  → 資金重新變成 ~50/50 配比（用 WBTC 和 USDC 重新部署）
  → BTC 繼續跌到 $82K → 再次累積 IL
  → 再次 rebalance → 再次鎖虧

第 3 輪 ... 第 964 輪：
  → 每一輪都在更低的價格重複同樣的循環
```

**每次 rebalance 等同於在下跌途中「抄底」**——vault 把剩餘的 USDC 換成更多 WBTC（因為要維持區間內的 50/50 配比），然後那些剛買的 WBTC 繼續貶值。

學術上這叫 **Loss-Versus-Rebalancing (LVR)**。在持續下跌的市場中，LVR 是集中流動性做市商最大的隱性成本。

### 5.4 三種策略的 IL 對比

| 策略 | IL 機制 | Jeff 的案例 |
|------|---------|-------------|
| **V2 全範圍（靜態）** | 被動承受，不放大 | ~1.3% ≈ $10 |
| **V3 集中（靜態不 rebalance）** | 一次性放大，跌出範圍就停 | ~20% 但不會更多 |
| **V3 集中 + 自動 rebalance（Steer）** | 每次 rebalance 鎖虧 + 重新曝險 | **12.7% ≈ $104**（964 次循環） |

實際 12.7% 低於理論單次 21.7%，是因為：
- 每次 swap 都有 0.05% 手續費收入（累積起來部分抵消 IL）
- 並非所有 964 次 rebalance 都發生在下跌中，有些是橫盤區間調整
- Merkl 獎勵額外補貼了一部分

## 六、「30% APR」為什麼沒兌現？

Vault 頁面顯示 **+30.24% Boost APR**。Jeff 可能預期：

```
$1,000 × 30% APR × (62 天 / 365 天) = +$51 收益
```

實際上他虧了 $185。差距來自以下認知落差：

| 預期 | 現實 |
|------|------|
| APR 是「淨收益率」 | APR 只計算 fee + 獎勵，**不含 IL** |
| BTC 價格不變 | BTC 跌了 26.7% |
| IL 很小（V2 直覺） | 集中流動性 IL 放大 10+ 倍 |
| Rebalance 是好事 | 下跌市場中 rebalance 加速虧損 |

**APR 的計算假設價格不變。** 一旦底層資產發生方向性移動，集中流動性的 IL 可以輕鬆超過全年的 fee 收入。

## 八、結論

Jeff 的 $185 虧損不是 vault 故障或策略 bug，而是三個因素疊加的必然結果：

1. **BTC 單邊下跌 26.7%**（佔虧損的 63%）——這是市場風險，任何持有 WBTC 的人都會承受
2. **集中流動性 + 頻繁 rebalance 放大了 IL**（佔虧損的 56%）——這是策略風險，Steer vault 在單邊行情中的結構性弱點
3. **KAT 獎勵只補回了一半 IL**（+$53 vs -$104）——激勵力度不足以覆蓋下跌行情中的 LVR 成本

核心教訓：**集中流動性 vault 的 rebalance 策略在橫盤市場很賺（高 fee 收入、低 IL），但在單邊趨勢中是結構性虧損的。** APR 數字不反映這個風險。
